# STM8 Modbus Air Sensor

Модуль мониторинга качества воздуха с интерфейсом **Modbus RTU (RS-485)**. Измеряет температуру, влажность, концентрацию CO₂, TVOC и рассчитывает индекс AQI. Предназначен для интеграции в системы умного дома, вентиляции и автоматизации зданий.

## Назначение

Устройство подключается к шине RS-485 и отдаёт показания сенсоров по протоколу Modbus RTU. Мастер-устройство (ПЛК, шлюз, контроллер) опрашивает датчик, считывая регистры с измеренными значениями.

**Измеряемые параметры:**

| Параметр | Сенсор | Единица |
|----------|--------|---------|
| Температура | AHT30 | °C (×10) |
| Влажность | AHT30 | % (×10) |
| eCO₂ | ENS160 | ppm |
| TVOC | ENS160 | ppb |
| AQI | ENS160 | 1–5 |
| Статус AHT30 | — | 0 = OK |
| PART_ID ENS160 | — | 0x0160 |
| DATA_STATUS ENS160 | — | статус |

## Аппаратная часть

### Компоненты

| Устройство | Интерфейс | Назначение |
|------------|-----------|------------|
| **STM8S103C8T6** | — | Микроконтроллер (16 MHz, 8 KB Flash, 1 KB RAM) |
| **AHT30** | I²C (адрес 0x38) | Внешний датчик температуры и влажности |
| **ENS160 + AHT21** | SPI | Плата: eCO₂/TVOC/AQI + встроенный датчик T&H |
| **MAX485** | UART | Трансивер RS-485 для Modbus RTU |


### Подключение выводов

| № | GPIO | Функция | Подключение | Примечание |
|---|------|---------|-------------|------------|
| 2 | PD5 | UART1_TX | → MAX485 DI | Modbus TX |
| 3 | PD6 | UART1_RX | ← MAX485 RO | Modbus RX |
| 4 | PD1 | BUTTON- | → GND | Кнопка (выход LOW) |
| 5 | PA2 | LED | → 3.3V через 330Ω | Индикация |
| 6 | PA3 | SPI_CS | → ENS160 CS | Chip Select |
| 11 | PB5 | I2C_SDA | → AHT30 SDA | **Pull-up 4.7kΩ к 3.3V** |
| 12 | PB4 | I2C_SCL | → AHT30 SCL | **Pull-up 4.7kΩ к 3.3V** |
| 15 | PC5 | SPI_SCK | → ENS160 SCK | |
| 16 | PC6 | SPI_MOSI | → ENS160 SDI | |
| 17 | PC7 | SPI_MISO | ← ENS160 SDO | |
| 20 | PD3 | BUTTON+ | → кнопка → PD1 | Вход с pull-up |

### Схема подключения RS-485

```
STM8S           MAX485
------          ------
PD5 (TX)  -----> DI
PD6 (RX)  <----- RO
3.3V      -----> DE + RE
GND       -----> GND
                  A ──────> Modbus A+
                  B ──────> Modbus B-
```

### Схема подключения датчиков

```
I²C (AHT30 — внешний датчик):
  PB4 (SCL) ───┬────> AHT30 SCL
               └───[4.7kΩ]─── 3.3V
  PB5 (SDA) ───┬────> AHT30 SDA
               └───[4.7kΩ]─── 3.3V

SPI (ENS160):
  PA3 (CS)   ──────> ENS160 CS
  PC5 (SCK)  ──────> ENS160 SCK
  PC6 (MOSI) ──────> ENS160 SDI
  PC7 (MISO) ──────> ENS160 SDO
  3.3V       ──────> ENS160 VDD
  GND        ──────> ENS160 GND
```

### Кнопка и светодиод

```
Кнопка:
  PD1 ──── GND (выход LOW)
  PD3 ────[КНОПКА]──── PD1

Светодиод:
  PA2 ────[330Ω]────> LED ──── 3.3V
```

## Modbus RTU

### Параметры связи

| Параметр | Значение |
|----------|----------|
| Скорость | 9600 бод |
| Данные | 8 бит |
| Стоп | 1 бит |
| Чётность | Нет |
| Адрес по умолчанию | 1 |

### Карта регистров

| Адрес | Название | Описание | Диапазон | Ед. изм. |
|-------|----------|----------|----------|----------|
| 0 | TEMP | Температура (AHT30) | -500..+1000 | °C ×10 |
| 1 | HUMID | Влажность (AHT30) | 0..1000 | % ×10 |
| 2 | ECO2 | Эквивалент CO₂ (ENS160) | 0..65535 | ppm |
| 3 | TVOC | Летучие органические соединения | 0..65535 | ppb |
| 4 | AQI | Индекс качества воздуха | 1..5 | — |
| 5 | STATUS_AHT30 | Статус AHT30 | 0=OK, >0=ошибка | — |
| 6 | PART_ID | ID чипа ENS160 | 0x0160 | — |
| 7 | DATA_STATUS | Статус данных ENS160 | см. даташит | — |

> **Важно:** Температура и влажность (регистры 0–1) умножены на 10. Для получения реального значения разделите на 10. Пример: `242` → `24.2°C`.

#### Чтение всех регистров сразу (0–7)

**Запрос:**
```
01 03 00 00 00 08 44 2F
```

**Ответ:**

## Конфигурирование Modbus-адреса

Устройство поддерживает ручную настройку адреса (1–247) через кнопку.

### Как изменить адрес

1. **Удерживайте кнопку 3 секунды** → светодиод мигнёт 2 раза (вход в режим конфигурации)
2. **Короткое нажатие** (< 0.5 сек) → адрес **+1**
3. **Среднее нажатие** (0.5–1.5 сек) → адрес **−1**
4. **Длинное нажатие** (> 1.5 сек) → **сохранить и выйти** (3 быстрых мигания)

После каждого нажатия светодиод моргнёт текущим значением адреса.

### Первый запуск

Если EEPROM пуст (новое устройство), модуль **автоматически войдёт в режим конфигурации** при первом включении.

## LED-индикация

| Сигнал | Описание |
|--------|----------|
| Короткая вспышка | Приём Modbus-команды |
| N морганий | Текущий адрес = N |
| 2 быстрых мигания | Вход в режим конфигурации |
| Мигание раз в 1 сек | Режим конфигурации активен |
| 3 быстрых мигания | Адрес сохранён |
| Нет света | Ожидание / нет активности |
